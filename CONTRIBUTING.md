# Contributing to MDLBEAST Archive System

Thank you for your interest in contributing to the MDLBEAST Archive & Communications System.

This is a **proprietary software project** developed exclusively for MDLBEAST Entertainment Company. 

## Contribution Policy

Due to the proprietary nature of this software:

1. **External Contributions**: Not accepted unless by prior written agreement
2. **Code Access**: Limited to authorized developers only
3. **Licensing**: All code remains property of Mahmoud Fouad
4. **NDA Required**: Contributors must sign non-disclosure agreements

## For Authorized Contributors

If you have been granted authorization to contribute:

### Getting Started
1. Contact the project maintainer for access credentials
2. Review and sign the contributor agreement
3. Set up your development environment following the README

### Development Workflow
1. Create a feature branch from `main`
2. Follow the existing code style and conventions
3. Write meaningful commit messages
4. Test your changes thoroughly
5. Submit for code review before merging

### Code Standards
- Use TypeScript for type safety
- Follow the existing project structure
- Write clear documentation
- Include tests for new features
- Follow security best practices

### Commit Message Format
```
type(scope): subject

body (optional)

footer (optional)
```

Types: `feat`, `fix`, `docs`, `style`, `refactor`, `test`, `chore`
